# OWASP 团队分享任务清单

## 目标

在团队内部完成一次面向 Web 开发团队的 OWASP 安全编码规范分享，并交付以下成果：

- `原始材料.md`：四大主题的原始资料、案例、想法、创意整理
- `分享资料.md`：可支撑约 1 小时分享的讲稿型材料
- `slidev.md`：用于生成演示文稿的 Slidev 源文件
- 最终演示文稿：基于 Slidev 生成，布局美观、结构清晰、适合内部分享

## 阶段一：收集并沉淀原始材料

交付物：`原始材料.md`

- [ ] 建立 `原始材料.md` 文档骨架，按四个主题分章节
- [ ] 通读并提炼 `OWASP_zh.pdf` 的重点注意事项
- [ ] 从日常 Web 开发视角筛选高相关内容
- [ ] 按角色拆分注意点：前端、后端、运维/部署
- [ ] 结合团队技术栈补充落地建议
说明：重点覆盖 `Java`、`Spring Boot`、`MySQL`、常见中间件、认证授权、日志、配置、文件上传、接口安全、依赖管理等场景
- [ ] 收集“OWASP Top 10”相关材料
说明：需优先核实 2025 年是否存在官方新版 Top 10；若无，则以官方现行版本为主，并补充 2025 年仍高频的风险、案例与防范建议
- [ ] 为 Top 10 每一项整理以下内容
说明：风险定义、典型攻击方式、常见触发场景、对团队的实际影响、在当前技术栈下的防范措施
- [ ] 收集 AI 时代 OWASP 的变化与新增方向
说明：至少包含 `OWASP Agentic Skills Top 10`，同时继续发掘与 AI Agent、LLM 应用、安全治理、供应链、模型使用边界相关的 OWASP 新项目或倡议
- [ ] 梳理“AI 如何增强 OWASP 落地”的方案池
说明：至少包含 `OWASP agent skill` 方案，并扩展其他可行方向，例如需求评审、架构评审、代码审查、CI/CD 检查、安全基线生成、开发辅助问答、培训演练等
- [ ] 为每个主题记录可直接用于演讲的案例、类比、故事化表达
- [ ] 为每个主题整理官方来源与参考链接
- [ ] 完成 `原始材料.md` 初稿
- [ ] 对 `原始材料.md` 做一次去重和结构整理

### 阶段一验收标准

- [ ] 四个主题都有独立章节
- [ ] 每个主题都包含“是什么、为什么重要、我们团队会怎么遇到、怎么防”
- [ ] 材料中明确区分“官方结论”“团队场景化理解”“可执行建议”
- [ ] 关键结论均能追溯到权威来源

## 阶段二：整理为可讲 1 小时的分享资料

交付物：`分享资料.md`

- [ ] 设计整场分享的主线结构
说明：建议形成“基础规范 -> Top 10 风险 -> AI 时代变化 -> AI 驱动落地”的叙事顺序
- [ ] 规划 1 小时内容配比
说明：建议预留开场 5 分钟、主体 45 至 50 分钟、总结与互动 5 至 10 分钟
- [ ] 从 `原始材料.md` 中筛选高价值内容，删除冗余细节
- [ ] 将原始条目改写为适合口头表达的讲稿材料
- [ ] 为每个章节补充过渡语与总结语
- [ ] 增加与团队实际工作相关的示例
说明：接口鉴权、SQL 注入、XSS、CSRF、越权、敏感信息泄露、依赖漏洞、文件上传、配置错误、日志审计、密钥管理、部署暴露面等
- [ ] 为 AI 时代部分形成清晰观点
说明：不仅介绍 OWASP 新变化，还要说明“为什么这和我们现在的研发方式相关”
- [ ] 为“OWASP agent skill”形成较具体方案
说明：明确使用场景、输入输出、接入时机、预期收益、限制条件
- [ ] 增补其他 AI 增强方案的比较
说明：哪些适合快速试点，哪些适合长期建设
- [ ] 控制篇幅，确保总内容量可支撑约 1 小时
- [ ] 完成 `分享资料.md` 初稿
- [ ] 以“演讲者视角”通读一遍并压缩不必要内容

### 阶段二验收标准

- [ ] 结构完整，逻辑顺畅，可直接作为演讲底稿
- [ ] 重点明确，不堆砌概念
- [ ] 对团队角色有针对性，不是泛泛而谈
- [ ] 预计口述时长约 55 至 65 分钟
